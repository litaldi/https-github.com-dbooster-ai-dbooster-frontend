### Font Weights
- **Regular (400)**: Body text and descriptions
- **Medium (500)**: Form labels and card titles  
- **Semibold (600)**: Section headings
- **Bold (700)**: Page titles and emphasis

## Spacing Scale

Consistent spacing using a 4px base unit:

- **xs**: 4px (0.25rem)
- **sm**: 8px (0.5rem)  
- **md**: 16px (1rem)
- **lg**: 24px (1.5rem)
- **xl**: 32px (2rem)
- **2xl**: 48px (3rem)

## Component Guidelines

### Buttons
- Use `primary` for main actions
- Use `secondary` for supporting actions
- Use `destructive` for dangerous actions
- Always provide adequate touch targets (minimum 44px)

### Forms
- Always pair inputs with labels
- Use placeholder text sparingly
- Provide clear error messages
- Group related fields logically

### Cards
- Use for grouping related content
- Maintain consistent padding
- Use CardHeader for titles and descriptions
- Place actions in CardFooter

## Accessibility

All components follow WCAG 2.1 AA guidelines:

- **Color Contrast**: Minimum 4.5:1 for normal text, 3:1 for large text
- **Keyboard Navigation**: All interactive elements are keyboard accessible
- **Screen Reader Support**: Proper ARIA labels and semantic HTML
- **Focus Management**: Clear focus indicators and logical tab order

## Responsive Design

Our components are mobile-first and responsive by default:

- **Mobile**: 375px and up
- **Tablet**: 768px and up  
- **Desktop**: 1024px and up
- **Large Desktop**: 1280px and up

Use Tailwind's responsive prefixes (`sm:`, `md:`, `lg:`, `xl:`) for responsive behavior.
